# 文章编辑器：多平台发布与模板管理设计方案（v3）

## 一、竞品分析

### 1.1 多平台发布工具

| 工具 | 核心做法 | 样式转换能力 | 对本案的参考 |
|------|---------|-------------|-------------|
| **135 编辑器** | 微信排版最深度。多平台发分发靠生成长图或复制粘贴，不做格式转换 | ❌ 其他平台仅贴图 | 非微信平台不需要 HTML 样式转换 |
| **Wechatsync** | 微信文章格式 → 其他平台保留基础格式（加粗/引用/列表） | 有限，基础 Markdown | 非微信平台走降级策略 |
| **TurboPush** | Markdown 编辑器 → 多平台 API 直推，内置 doocs/md 转微信 HTML | 微信用 MD→HTML，其他平台简化 | 以 Markdown 为中间格式的思路 |
| **MultiPost** | 同一内容 → AI **重写**为各平台风格文案 | 内容改写，非样式转换 | 样式转换不是好方向 |
| **content-pipeline** | 公众号排版（135 式） → 小红书生成轮播图（用 CDP 自动化操作浏览器发布） | 微信→小红书是**图片**，非 HTML | 小红书只接受图片 |
| **MdNice / doocs/md** | Markdown → 微信 HTML，支持多主题 CSS | 标准化做法 | CSS 换肤思路可借鉴 |
| **知乎 WPL/s** | VS Code 插件，Markdown → 知乎文章 API 直推 | 只支持基础 Markdown | 知乎/掘金等平台只消费 MD |

### 1.2 各平台格式能力分级

| 平台 | 接收格式 | 富样式能力 | 交互式内容 | 策略 |
|------|---------|-----------|-----------|------|
| **微信公众号** | HTML（嵌套 section + 内联样式） | ✅ 最强 | ❌ 图表降级 | WeChatRenderer |
| **知识星球** | HTML（内联样式） | ✅ 中等 | ❌ | ZSXQRenderer |
| **问道 (SpaceMV)** | HTML + `{{chart:slug}}` `{{map:token}}` | ✅ 语义 HTML | ✅ ECharts 交互 | 保留语义 |
| **知乎** | HTML（基础 Markdown 风格） | ❌ 仅基础 | ❌ | 基础 HTML |
| **头条号/百家号** | HTML（基础） | ❌ | ❌ | 基础 HTML |
| **掘金/CSDN** | Markdown | ❌ | ❌ | 导出 Markdown |
| **小红书** | 图片轮播（3:4 竖版） | N/A | ❌ | 生成长图 |

### 1.3 竞品启示

1. **没有竞品做"同一篇文章在不同平台间转换 HTML 样式"**——非微信平台不支持 135 式复杂排版
2. **只有微信 + 知识星球需要富 HTML 渲染**
3. **知乎/掘金等开发者社区原生消费 Markdown**
4. **135 编辑器的核心体验是选中文字直接改颜色/字号/卡片风格**——不是从右侧面板调
5. **小红书只接受图片**，微信文章转小红书 = 生成轮播图

## 二、核心概念

### 2.1 角色分工

| 角色 | 职责 | 关注点 |
|------|------|--------|
| **创作者** | Word 写稿 → 导入编辑器 → 替图表为 `{{chart:slug}}` → 交稿 | 内容 |
| **运营** | 选模板 → 编辑 → **直接微调样式** → 预览多平台 → 发布 | 内容 + 排版效果 |
| **系统** | 存储 contentJson，按平台 Renderer 输出，调 API 推送 | 数据一致性 + 可扩展性 |

### 2.2 数据模型层级

```
层级 1: contentJson（ProseMirror 语义节点，唯一源）
  含可选的 styleOverrides（节点属性覆盖样式）

层级 2: stylePreset（全局样式预设）
  定义 h2/p/blockquote/chart 等节点的默认样式

层级 3: Renderer（平台适配器）
  contentJson + stylePreset → 平台原生格式
  渲染优先级: 元素 styleOverrides > stylePreset > Renderer 默认值
```

### 2.3 模板定义

**模板 = 内容骨架 + 样式预设名 + 默认平台**

```json
{
  "name": "格物科技深度 - 微信蓝",
  "platform": "WECHAT",
  "style_preset": "gewu-wechat-blue",
  "snapshot_json": {
    "type": "doc",
    "content": [
      { "type": "heading", "attrs": { "level": 2, "styleOverrides": null },
        "content": [{ "type": "text", "text": "文章标题" }] },
      { "type": "paragraph", "attrs": { "styleOverrides": null },
        "content": [{ "type": "text", "text": "正文内容..." }] }
    ]
  }
}
```

```

  └─ 将来: ...
```

## 四、核心模块详细设计

### 4.1 编辑器内直接改样式（核心能力）

#### 4.1.1 浮动工具栏

选中任意文本/块级元素时，弹出浮动工具栏：

```
选中 H2 → 浮动工具栏:
  ┌─ 颜色 ─┬─ 装饰 ─┬─ 字号 ─┬─ 操作 ─────┐
  │ [色块]  │ [左竖线]│ [20px] │ [编号01]  │
  │ [拾色器]│ [底线] │ [18px] │ [加粗]    │
  │         │ [色块] │ [22px] │ [清样式]  │
  │         │ [无]   │        │           │
  └─────────┴────────┴────────┴───────────┘
```

改了什么？→ 写入该节点的 `attrs.styleOverrides`：

```json
{
  "type": "heading",
  "attrs": {
    "level": 2,
    "styleOverrides": {
      "color": "#FF6D00",
      "borderLeft": "none",
      "borderBottom": "2px solid #FF6D00"
    }
  }
}
```

#### 4.1.2 清样式

浮动工具栏提供「清样式」按钮：清当前元素的 `styleOverrides`，还原为预设默认值。

#### 4.1.3 保存为新预设

```
改了一批元素样式后:
  点击「保存为新样式预设」
  ① 遍历 contentJson，收集所有非空 styleOverrides
  ② 合并 → 生成新的 StylePreset JSON
  ③ 弹窗输入名称 → 写入预设列表
  ④ 可选推送为团队共享预设
```

### 4.2 样式预设系统

#### 4.2.1 定义

```typescript
// styles/presets/gewu-wechat-blue.ts
export const gewuWechatBlue = {
  name: '格物微信蓝',
  platform: 'WECHAT',
  nodes: {
    h2:       { color: '#548DD4', fontSize: '20px', fontWeight: 'bold',
                borderLeft: '3px solid #548DD4', paddingLeft: '12px',
                marginTop: '24px', marginBottom: '12px' },
    h3:       { color: '#548DD4', fontSize: '17px', fontWeight: '600',
                borderLeft: '2px solid #548DD4', paddingLeft: '10px' },
    p:        { fontSize: '14px', lineHeight: '1.75', color: '#333', marginBottom: '10px' },
    blockquote: { bgColor: '#F6F9FD', borderRadius: '8px', padding: '16px',
                  borderLeft: 'none', margin: '16px 0' },
    chart:    { mode: 'placeholder' },
    img:      { borderRadius: '8px', maxWidth: '100%' }
  },
  global: {
    fontFamily: '-apple-system, "Microsoft YaHei", sans-serif',
    padding: '10px 16px'
  }
}
```

#### 4.2.2 渲染优先级引擎

```typescript
function getEffectiveStyle(nodeType, overrides, preset, rendererDefaults) {
  if (overrides) return { ...rendererDefaults, ...preset?.nodes?.[nodeType], ...overrides }
  if (preset?.nodes?.[nodeType]) return { ...rendererDefaults, ...preset.nodes[nodeType] }
  return rendererDefaults
}
```

#### 4.2.3 切换预设影响

```
从「格物微信蓝」→「格物快讯黑」:
  data-preset 属性改变 → CSS 换肤
  有 styleOverrides 的元素保留覆盖，不受影响
  无 styleOverrides 的元素跟随新预设
```

### 4.3 Renderer 系统

#### 4.3.1 定义

```typescript
interface Renderer {
  platform: string
  render(contentJson: object, options?: { stylePreset?: object }): string
}
```

#### 4.3.2 Renderer 列表

| Renderer | platform | 输出 | 说明 |
|----------|----------|------|------|
| WeChatRenderer | WECHAT | 135 式嵌套 `<section>` HTML | stylePreset 控制内联样式；styleOverrides 覆盖单元素 |
| WendaoRenderer | SITE_WENDAO | 语义 HTML + `{{chart:slug}}` | 忽略样式参数（样式由平台 CSS 控制） |
| ZSXQRenderer | ZHISHU_XINGQIU | 轻量 HTML（内联样式） | 类似 WeChat 但更轻 |
| ZhihuRenderer | 知乎 | 基础 HTML | 仅标题/加粗/引用/列表 |
| MDGenerator | 掘金/CSDN | Markdown | prosemirror-markdown 序列化 |
| XiaohongshuRenderer | XIAOHONGSHU | 3:4 竖版图片轮播 | 逐段渲染为图片 |

#### 4.3.3 WeChatRenderer 映射规则

| 语义节点 | stylePreset + styleOverrides → 输出 |
|----------|-----------------------------------|
| `h2` | `<section><section style="border-left:3px solid #548DD4;padding-left:12px"><h2 style="color:#548DD4;font-size:20px;font-weight:bold;margin:24px 0 12px">标题</h2></section></section>` |
| `h2` + styleOverrides `{color:"#FF0000"}` | `<section><section style="border-bottom:2px solid #FF0000"><h2 style="color:#FF0000;font-size:20px;font-weight:bold">标题</h2></section></section>` |
| `p` | `<p style="font-size:14px;line-height:1.75;color:#333;margin-bottom:10px">正文</p>` |
| `blockquote` | `<section style="background:#F6F9FD;border-radius:8px;padding:16px;margin:16px 0"><p>引用</p></section>` |
| `chartNode` | `<a href="https://..." style="display:block;border:1px solid #e8e8e8;border-radius:8px;padding:16px;text-align:center"><span style="color:#548DD4;font-weight:600">📊 查看完整图表 →</span></a>` |
| `img` | `<img src="..." style="border-radius:8px;max-width:100%">` |

### 4.4 预览机制

```
编辑器内: CSS 换肤模拟（速度优先，精度 90%）
  足够运营在创作时看到效果
  不处理 135 式嵌套 section

预览面板: 调后端 Preview API（精度 100%）
  POST /content/articles/{id}/preview
  Request: { platform: "WECHAT", stylePreset: "gewu-wechat-blue" }
  Response: { previewHtml: "完整 135 HTML 输出" }
  前端 iframe 渲染
  用于发布前最终确认
```

### 4.5 发布流程

```
运营点击 [发布]:
  ☑ 问道  ☑ 微信 (格物科技)  ☐ 知识星球

后端接收:
  POST /content/articles/{id}/publish/multi
  [
    { targetCode: "SITE_WENDAO" },
    { targetCode: "WECHAT_OFFICIAL_ACCOUNT", accountId: 1, stylePreset: "gewu-wechat-blue" }
  ]

PublishDispatcher 路由:
  WendaoPublisher:
    contentJson → WendaoRenderer → 语义 HTML + {{chart:slug}} → 发布

  WeChatPublisher:
    contentJson + stylePreset → WeChatRenderer → 135 HTML
    → convertInlineImages: 图片传微信 CDN + 入素材库
    → addDraft → publishDraft → 发布
```

  category        VARCHAR(50)  DEFAULT NULL COMMENT '模板分类',
  platform        VARCHAR(50)  NOT NULL COMMENT '默认目标平台',
  style_preset    VARCHAR(100) DEFAULT NULL COMMENT '默认样式预设名称',
  snapshot_json   LONGTEXT     NOT NULL COMMENT 'ProseMirror JSON 内容快照',
  thumbnail_url   VARCHAR(500) DEFAULT NULL COMMENT '缩略图',
  description     VARCHAR(500) DEFAULT NULL COMMENT '模板说明',
  sort_order      INT          DEFAULT 0 COMMENT '排序号',
  created_by      VARCHAR(64)  DEFAULT '' COMMENT '创建人',
  created_at      DATETIME     DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  del_flag        CHAR(1)      DEFAULT '0',
  INDEX idx_category (category),
  INDEX idx_platform (platform)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章模板';

## 七、工作流

### 7.1 创作者

```
Word 写稿 → 粘贴到编辑器 → 替换截图 → {{chart:slug}} / {{map:token}}
  → 提交审核/交给运营
```

### 7.2 运营

```
1. 选模板 → 新建文章，编辑器显示模板排版效果
2. 调整内容 + 微调样式（浮动工具栏直接改 + 右侧面板批量调）
3. [保存为新预设] 或 [切换预设预览不同平台]
4. 切换预览 tab 确认各平台效果
5. 选择目标 → 发布
```

| 环节 | 当前（手动） | 本方案 |

| 另发问道 | 未做或重复 | 1 次点击 |

## 九、关键设计决策

| 决策 | 结论 |
|------|------|
| 运营在哪里微调样式 | **编辑器内直接改（浮动工具栏），右侧面板也可批量调** |
| 改的是内容还是样式 | 改的是 styleOverrides，写入 contentJson，不影响其他元素 |
| 改完能不能批量应用 | **可以，保存为新样式预设** → 提取所有 styleOverrides → 新 StylePreset |
| 切换预设影响什么 | 没有 styleOverrides 的元素跟随新预设，有覆盖的不变 |
| 模板存什么 | snapshot_json（含 styleOverrides）+ style_preset_name + platform |
| 新增平台要多久 | 0.5-1 天（写一个 Renderer，纯配置级） |
| Markdown 平台怎么办 | 存 contentJson，用 prosemirror-markdown 导出，不返工 |
| {{chart}} 非问道平台 | 降级为可点击卡片链接或截图图片 |
| 和现有架构冲突吗 | 不冲突，纯增量。IContentPublisher 接口不变，ChartNode 不变，ContentArticle 不变 |
